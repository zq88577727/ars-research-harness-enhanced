## CI Finding

The first GitHub Actions run after P7-14 failed on commit
`03d35ba3daf518674783ae0e65bc09db45384407`. The failure reproduced in a clean
local checkout without ignored raw exports.

Root cause:

`validate_gbd_cvd_submission_readiness.py --require-manuscript-review-ready`
still required the local raw GBD source export, while P7-13/P7-14 intentionally
excluded that export from the public repository.

Fix:

- `validate_gbd_cvd_submission_readiness.py` now supports
  `--allow-missing-source-export`.
- `harness/scripts/run_all_validations.py` uses that public-release mode for
  the GBD CVD manuscript-review gate.
- Strict local validation can still run without the flag when the local export
  is present.

## Capability Boundary Audit

The public README, roadmap, and data policy should present these boundaries:

- NHANES: complete closed-loop teaching case.
- CHARLS: scaffold and local-only restricted-data adapter until provider access,
  official codebook/questionnaire files, and local raw files are available.
- GBD minimal demo: fixture-only / teaching route.
- GBD CVD instance: source-backed manuscript candidate with query, provenance,
  derived summary, citation/reuse, and claim-review gates; not submission-ready;
  raw source export is local-only and not redistributed.

## P7-15 Decision

The public release is safe to show only after the CI fix commit is pushed and
the new GitHub Actions run is green. The raw GBD CVD source CSV is not present
in the remote tree and remains local-only.
